VCS Change History

Changes in VCS 0.6

Perforce: Filenames containing @, %, # and * should now be properly encoded and decoded.

Git: Improved support for older versions of Git and fixed vcs status.

Darcs: Tests now set $EMAIL to avoid a interaction (or a hang!)

Mercurial: Improved support for older versions.

Changes in VCS 0.5

New environment variables VCS_PAGER and VCS_DIFF_PAGER allow you to feed log and diff output through colorizers and into pagers automatically. See the man page for details.

Changes to Perforce support

Perforce support is now more self-consistent. In particular vcs diff and vcs commit now have a default of ..., applying them to just the current directory and its children rather than the whole workspace.

vcs status now produces much more useful output for Perforce, and honors .vcsignore files.

Changes in VCS 0.4

New version control systems supported:

  • Mercurial
  • Darcs
  • Perforce

New commands:

  • vcs annotate - annotate lines with revision numbers
  • vcs clone - make a local copy from a repository

VCS now depends on cURL.

Fix a bug where the detection of the underlying version control system could go into an infinite loop.

Changes in VCS 0.3

Git is now supported.

VCS now includes a simple test suite.

Changes in VCS 0.2

New commands:

  • vcs status - summarize current status
  • vcs update - update to head revision
  • vcs log - display change history

Other sources of change information:

  • bzr changelog for development branch
  • bzr log in a clone of http://www.greenend.org.uk/rjk/bzr/vcs.dev
